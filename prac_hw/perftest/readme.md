# Тестер производительности (latency и throughput, CPE)

Требуется реализовать тестер производительности, который измеряет **latency** и **throughput** в единицах **CPE** (cycles per element), с устойчивыми результатами.

## Решение

Реализация в [perftest.cpp](perftest.cpp).

Замер длительности участка кода производится в тактах TSC (`rdtsc` / `rdtscp`, сериализация через `lfence`). На архитектурах, отличных от x86/x86_64, сборка завершается с `#error`. Чтобы снизить разброс из-за миграций между ядрами на **Linux** поток закрепляется на **CPU 0** (`pthread_setaffinity_np`).

Перед измерением проводится прогрев кеша без записи статистики (`warmup_runs` прогонов). Затем в `measure_runs` независимых замеров: считается CPE = `cycles / ops_per_run`, а затем **медиана** (*median*) , **среднее арифметическое** (*mean*), **стандартное отклонение** (*stddev*), **коэффициент вариации** (*cv_percent*) для всех CPE. Для **Latency** измеряется одна зависимая цепочка `x = f(x + 1)` на каждой итерации; для **Throughput** же `THROUGHPUT_LANES` независимых цепочек вызовов `f`.



## Сборка и запуск

```shell
cd prac_hw/perftest
make
./perftest
```

## Результаты эксперимента

Для демонстрации взята функция `sqrt` в виде `[](float x) { return std::sqrt(x); }`
Параметры замеров: `ITERS = 20'000'000`, `WARMUP_RUNS = 4`, `MEASURE_RUNS = 13`, `THROUGHPUT_LANES = 8`.

```text
Performance test
iters=20000000, warmup_runs=4, measure_runs=13, throughput_lanes=8

Latency (CPE):
	samples_cpe = [13.649648, 13.639154, 13.637227, 13.641730, 13.644254, 13.646536, 13.629241, 13.631505, 13.653141, 13.666948, 13.642106, 13.629586, 13.635788]
	median_cpe  = 13.641730
	mean_cpe    = 13.642066
	stddev_cpe  = 0.010104
	cv_percent  = 0.074067%

Throughput (CPE):
	samples_cpe = [3.900091, 3.898765, 3.898393, 3.894657, 3.901000, 3.899562, 3.898300, 3.896166, 3.900740, 3.897729, 3.898920, 3.900342, 3.897471]
	median_cpe  = 3.898765
	mean_cpe    = 3.898626
	stddev_cpe  = 0.001760
	cv_percent  = 0.045151%
```

Отношение **median latency** к **median throughput** как самых стабильных к выбросам метрик при `THROUGHPUT_LANES = 8`: **13.641730 / 3.898765 ~= 3.50**. Низкий `cv_percent` (0.05-0.08%) показывает, что разброс результатов небольшой и замеры стабильны.

Сравнение двух режимов показывает разницу между задержкой одной зависимой операции `f` и пропускной способностью при заданном числе независимых цепочек `THROUGHPUT_LANES`
