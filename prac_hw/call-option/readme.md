# Цена колл опциона Европейского типа

Требуется оценить цену колл опциона Европейского типа методом **Монте‑Карло** и показать корректность реализации.

Параметры из задания:

| Параметр | Значение |
|----------|----------|
| Безрисковая ставка $r$ | $0.05$ |
| Волатильность $\sigma$ | $0.1$ |
| Спот $S_0$ | $100$ |
| Страйк $K$ | $100$ |
| Срок $T$ | $1$ |
| Число траекторий $N$ | $1\,000\,000$ |

## Теория

Теоретическая цена:

```math
C_T = e^{-rT}\,\mathbb{E}\bigl[(S_T - K)^+\bigr],\qquad x^+ = \max(0,x).
```

Оценка Монте‑Карло:

```math
\hat{C}_T = e^{-rT}\,\frac{1}{N}\sum_{j=1}^{N}\bigl(\tilde{S}_{j,T} - K\bigr)^+.
```

Дискретное приращение записывается с дрейфом $r$:

```math
\tilde{S}_{t_i} = \tilde{S}_{t_{i-1}}\bigl(1 + r\,\delta + \sigma\sqrt{\delta}\,X_i\bigr),\quad X_i\sim N(0,1).
```

В реализации основной эксперимент использует точную выборку $S_T$ при логнормальном законе:

```math
S_T = S_0\exp\Bigl(\bigl(r - \tfrac{\sigma^2}{2}\bigr)T + \sigma\sqrt{T}\,Z\Bigr),\quad Z\sim N(0,1).
```

Для **проверки корректности** вычисляется аналитическая цена **Блэка-Шоулза**:

```math
C_T = S_0\,\Phi(y_+) - K e^{-rT}\,\Phi(y_-),
```

```math
y_\pm = \frac{\ln(\frac{S_0}{K}) + T\,(r \pm \sigma^2/2)}{\sigma\sqrt{T}},
```

где $\Phi$ - функция распределения стандартного нормального закона.

Погрешность метода Монте‑Карло по статистике убывает как $\frac{1}{\sqrt{N}}$, при $N=10^6$ типичный масштаб ошибки оценки - порядка $10^{-3}$.

## Решение

Реализация представлена в скрипте [call_option.py](call_option.py) с использованием библиотек `numpy` и `matplotlib` для визуализации. В качестве генератора использован `numpy.random.Generator` (PCG64).

Дополнительно выводится иллюстрация **явной схемы Эйлера** с малым числом шагов и траекторий. У такой схемы есть смещение порядка $O(\delta)$ относительно непрерывной модели.

## Эксперимент

При сравнении с аналитикой при $N=10^6$ оценка $C_T$ и аналитический $C_{BS}$ должны отличаться примерно на величину стандартной ошибки оценки. Для демонстрации сходимости по $N$ значений строится график:
```math
|\hat{C}_T - C_{\mathrm{BS}}|
```
Ожидается согласование с масштабом $\frac{1}{\sqrt{N}}$.

График сохраняется в `convergence.png` после запуска.

# Пакет опционов

Реализован расчёт пакета из $n=100$ колл‑опционов со случайно сгенерированными параметрами $(S_0, K, r, \sigma, T)$ и $N=100\,000$ траекторий. Используется точная дискретизация конечного значения $S_T$, как в основной реализации. Цикл по опционам распараллелен через **OpenMP** (`parallel for`): каждый поток обрабатывает свой набор индексов опционов и ведёт собственный буфер под блок случайных чисел. В качестве векторизации внутри каждого опциона траектории обрабатываются блоками размера `BLOCK_SIZE` (256), а после заполнения блока стандартными нормальными величинами сумма считается циклом с директивой `omp simd reduction`.

## Векторизация расчёта опционов

Генератор `std::normal_distribution` выдаёт скаляр, последующий этапом идёт вычисление $S_T = S_0\exp(\mu + \sigma\sqrt{T}\,Z)$ и $\max(S_T-K,0)$, что хорошо укладывается в  цикл по массиву $Z$ фиксированной длины. Размер блока (`BLOCK_SIZE`) задан явно, чтобы держать рабочие массивы в L1/L2 и не раздувать стек случайными окнами произвольной длины, а также дать компилятору предсказуемую длину внутреннего цикла для **автовекторизации** или `omp simd`.

Параллелизм реализован в виде разделения исполнения на разные OpenMP потоки, обрабатывающие разные независимые опционы. Внутри каждого опциона реализован цикл по индексам в блоке с `reduction` суммирует `payoff` и компилятор может сопоставить итерации с векторными регистрами.

Для сопоставимости с аналитикой Блэка-Шоулза используются стандартные `std::exp` / `std::erf` без `-ffast-math`.
