# Оптимизация матричного умножения

Требуется оптимизировать вычисление `C = A * B` (без BLAS) для размеров матриц, укладывающихся в L1 кэш, и сравнить базовую и оптимизированную реализации.

## Описание решения

Реализованы 2 версии умножения матриц $A[m \times k] * B[k \times n]$:

### 1. Наивное перемножение
```math
C_{ij} = \sum_{p=0}^{k-1} A_{ip} \cdot B_{pj}
```
Доступ к $B_{pj}$ осуществляется по столбцу, что нарушает пространственную локальность данных и вызывает промахи в кеше.

### 2. Оптимизированное умножение
В основе оптимизации лежит повышение пространственной локальности данных за счет построчного перемножения на $B^T$, умножение выполняется блоками по индексам $i, j, k$. Это снижает число промахов в кеше и ускоряет вычисления.

## Конфигурация эксперимента

Проверяются размеры степеней двойки, включая пример из задания:

|$A$|`32x64`| `64x64` |  `128x64` |`128x128` |
|-|-|-|-|-|
|$B$|`64x16` |`64x64` |  `64x64` | `128x128` |

В качестве метрики производится подсчет минимальное количество тактов из нескольких запусков и подсчитывается коэффициент ускорени: 

> `speedup = naive_cycles / blocked_cycles`

В качестве метрики совпадения результата считается Max-норма по разнице матриц:

> `max_abs_diff = max |C_naive - C_blocked|`.

## Сборка и запуск

```shell
cd prac_hw/matrix
make
./matrix
```

Параметры сборки: `-O3 -march=native -std=c++17`
