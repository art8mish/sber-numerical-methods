# Домашнее задание: векторный `minstd_rand` (fp32) и параллельный Monte Carlo для $\pi$

Реализация соответствует формулировке из методички:

## Задача 1 — векторная генерация fp32

- Язык **C++**, тип выхода **`float` (fp32)** для метрик производительности и массовой генерации U[0,1).
- Целочисленное ядро ЛКГ то же, что у **`std::minstd_rand`** ($a=48271$, $c=0$, $m=2^{31}-1$); своё умножение по модулю Мерсенна **без сторонних ГСЧ**.
- Векторизация: блок из **8** последовательных выходов через предвычисленные $a^k \bmod m$ и преобразование в fp32 с помощью **AVX2** (`__m256`), автосветилляция циклов возможна для скалярного варианта.
- Длины буферов — **степени двойки** (в тестах $2^{16}$, $2^{24}$).
- **Проверка корректности:** поток `uint32_t` сравнивается с **`std::minstd_rand`** на $2^{20}$ шагов; массив fp32 из скалярной и векторной генерации на $2^{16}$ элементов — побитово совпадает.
- **Производительность:** в stdout выводится **median clocks на элемент** для скалярной и векторной генерации U[0,1) ($N=2^{24}$, 7 прогонов).

## Задача 2 — параллельный расчёт $\pi$

- Параллелизм: **OpenMP**.
- Используется тот же ЛCG и **батчевая схема $a^k \bmod m$** по **8** числам на поток (аналог векторной генерации из задачи 1); затем перевод в $\mathrm U(-1,1)$ в fp32 тем же правилом, что и в последовательном коде.
- Потоки разбивают диапазон точек; стартовое состояние ГСЧ для отрезка задаётся через **прыжок** $A^{2\cdot\text{start}} \cdot s_0 \pmod m$ (один глобальный поток так же, как у последовательного варианта).
- Объём: **$10^8$ двумерных точек** (пара случайных чисел на точку).

## Дополнительно (лекции)

Сохранён бенчмарк **double**: скалярный vs блочный $\pi$ для $N=20\,000\,000$ и подбор `BLOCK_SIZE` из $\{8,16,32,64,512\}$.

## Сборка и запуск

```shell
cd prac_hw/minstd-rand
make
./minstd_rand
```

Флаги: `-O3 -Wall -Wextra -std=c++17 -march=native -fopenmp`.

Опции:

- `--verify-only` — только проверки совпадения с `std::minstd_rand` и fp32 scalar/vector.
- `--bench-double` — только double-бенчмарк.

## Пример вывода (зависит от CPU)

Указаны типичные строки: проверки **PASS**, **clocks/element** для fp32 scalar/vector, **parallel pi** с `clocks/point` по `rdtsc` и оценка $\pi$.

## Вычислительное окружение (пример)

- Ядро: Linux 6.8.x  
- Сборка: GCC с поддержкой **OpenMP** и **AVX2** (`-march=native` на целевой машине).
