# Оптимизация `minstd_rand` для Монте-Карло

Требуется реализовать генератор `minstd_rand` (Park-Miller LCG) и сравнить два варианта использования в задаче оценки числа `pi` методом Монте-Карло:

- последовательная генерация по одному числу;
- блочная генерация через предвычисленные степени `a^k mod m`.

Параметры генератора из лекции:

- `x_{i+1} = a * x_i mod m`;
- `a = 48271`;
- `m = 2^31 - 1 = 2147483647`.

## Идея оптимизации

Для базового варианта используется обычная рекуррентная схема, где каждое следующее значение зависит от предыдущего.

В оптимизированной версии формируется блок значений:

`x_{i+j} = a^j * x_i mod m`, `j = 1..k`.

Это позволяет генерировать сразу `k` чисел из одного базового состояния и уменьшить накладные расходы в горячем цикле Монте-Карло.

Также применена быстрая редукция по модулю Мерсенна `m = 2^31 - 1`:

`z = (z >> 31) + (z & m)` с финальной нормализацией.

## Что делает программа

Файл `minstd_rand.cpp`:

- реализует генератор в простом процедурном стиле (без сложной объектной иерархии);
- переводит целые состояния в `U(-1, 1)`;
- оценивает `pi` по `N = 20_000_000` точек;
- меряет время в тактах CPU (`rdtsc`, минимум из нескольких запусков);
- выводит `pi`, число тактов и ускорение.

## Сборка и запуск

```shell
cd prac_hw/minstd_rand
make
./minstd_rand
```

Параметры сборки: `-O3 -Wall -Wextra -std=c++17 -march=native`.
