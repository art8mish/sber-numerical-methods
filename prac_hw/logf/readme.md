
# Реализация ln

Требуется имплементировать float функции натурального логарифма ln с точностью <3.5 ulp в режиме округления `TO NEAREST` на С. Диапазон аргумента $2^{-126} \leq x < 2^{127}$ (положительные нормализованные числа; для субнормалей реализация также определена). Решение должно иметь вид:

```C
float logf (float x) {
    return res;
}
```

Решение должно быть корректным на всем указанном диапазоне аргументов. Использовать функции сторонних библиотек, например gnu libc, нельзя.

## Описание решения

Для $x > 0$ в формате IEEE-754 single разбиваем $x = 2^e \cdot r$, где $r \in [1, 2)$ - мантисса после нормализации, $e$ - целый порядок. Для субнормального $x$ сначала сдвигаем биты влево, пока не появится ведущая единица, поправляя $e$ (стандартная нормализация). Подствляя $x$: $ln x = e \cdot ln 2 + ln r$. Константа $ln 2$ задана как `double` с полной точностью для промежуточных вычислений.

Для вычисления $ln r$ используется подстановка $u = \frac{(r − 1)}{(r + 1)}$, то есть $ln r = 2 \cdot atanh(u)$. Чтобы уменьшить ошибку аппроксимации, перед полиномом выполняется дополнительная редукция диапазона: при $r > \sqrt{2}$ берётся $r \leftarrow r/2$ и $e \leftarrow e+1$. После этого мантисса лежит ближе к 1, что заметно повышает точность на точках около 1.

Скалярные выражения считаются в `double`, итог приводится к `float` (округление к ближайшему по правилам IEEE). Так достигается запас по сравнению с требованием **< 3.5 ulp** без вызова `libm`.

## Особые случаи

При $x \leq 0$ (включая $−0$): для нуля возвращается $-\infty$ с `errno = ERANGE`, `FE_DIVBYZERO`. Для отрицательного значения возвращается NaN с `errno = EDOM`, `FE_INVALID` (как при вызове `log` из libm). Значения $+\infty$ и NaN возвращаются как есть, без установки `errno`.

## Сборка

Команда `make` в каталоге `logf` собирает объектный файл `logf.o` (исходник - C++ для удобства, точка входа объявлена как `extern "C" float logf(float)` для связки с кодом на C).

## Тестирование

Реализация представлена в файле [test_logf.cpp](test_logf.cpp). Эталон - `std::log(double)` (сборка с `-fno-builtin-log` и `-fno-builtin-logf`). Проверяются `errno` и маска `FE` как у libm на особых аргументах ($+0$, $−0$, $−1$, $-\infty$, $+\infty$, NaN и отрицательные субнормали). На интервалах по битам положительных конечных чисел проверяются $ULP \leq 4$ и `errno == 0` после `logf`. Проводятся явные проверки точек и плотный диапазон вокруг 1.0, чтобы не пропускать локальные ULP‑пики. 

Запуск: `make test`

## Оптимизации

Для задания на **throughput** добавлена векторная реализация ядра на **AVX2 + FMA** (256 бит): [logf_avx2.cpp](logf_avx2.cpp), точка входа `__m256 logf8_avx2(__m256 x)` — **8** значений `float` за вызов.

**Алгоритм** тот же, что у скалярного `logf` в [logf.cpp](logf.cpp): разложение $x = 2^e \cdot r$ с мантиссой $r \in [1, 2)$, дополнительная редукция при $r > \sqrt{2}$ (деление $r$ на 2 и увеличение $e$), затем $\ln r = 2 \cdot \mathrm{atanh}(u)$, $u = (r-1)/(r+1)$, развёрнутый Горнер по $p = u^2$ с коэффициентами $1/(2k+1)$ в **double** и финальное $\ln x = e \cdot \ln 2 + \ln r$ с приведением к `float`.

**Оптимизации под SIMD:**

- **без ветвлений** по ланям: условие $r > \sqrt{2}$ через `_mm256_cmp_ps` + `_mm256_blendv_ps` для мантиссы; для порядка `e -= mask` (маска лана = `-1`, поэтому вычитание это `+1`);
- **FMA** в развёрнутом Горнере для полинома по $p$ (`_mm256_fmadd_pd`);
- разбиение регистра на две половины по 4 ланам, конверсии `cvtps_pd` / `cvtpd_ps` и `cvtepi32_pd` — полином и финальная сборка в `double`, что даёт согласованную с скаляром точность.

**Область определения быстрого ядра:** по условию оптимизации рассматриваются только **нормализованные** положительные конечные числа (biased exponent в $[1,254]$). Субнормали, нули, отрицательные, $\pm\infty$ и NaN в векторном ядре **не обрабатываются** (результат не определён); полный контракт как у `logf` остаётся в скалярной реализации.

**Сборка:** `g++ -std=c++20 -O2 -mavx2 -mfma -c logf_avx2.cpp` (в `make all` собирается вместе со скаляром).

### Проверка векторной версии

Тест: [test_logf_avx2.cpp](test_logf_avx2.cpp). Запуск: `make test_avx2` (целиком — `make check`).

Принципы:

- **CPUID-гард**: тест собирается **без** глобальных AVX2 флагов; AVX-путь обёрнут в `__attribute__((target("avx2,fma")))` и вызывается только после `__builtin_cpu_supports`. Если расширений нет на CPU — печатается `SKIPPED` и возвращается код **0**, чтобы `make check` отрабатывал на любой машине.
- **Эталон** — `std::log(double)`, сборка с `-fno-builtin-log` и `-fno-builtin-logf`; ULP считается относительно ближайшего `float` к эталону.
- **Область** покрытия: нормализованные положительные конечные числа. Набор «интересных» точек (`1`, $\pm 1\,\text{ULP}$ вокруг 1, $\sqrt{2}\pm$, $e$, степени 10, `FLT_MIN`, `FLT_MAX`), плотный скан вокруг 1.0 и $\sqrt2$ и 4 бакета по диапазону нормализованных. Лансы добиваются повторением последнего валидного значения, чтобы все 8 ланей всегда были в области.
- **Критерий**: `max_ulp ≤ 4` (тот же порог, что в [test_logf.cpp](test_logf.cpp)). В выводе печатаются `checked`, `bad`, `max_ulp` и худшая точка.

**Результат на Ryzen 5 5600H:** `checked=45016 bad=0 max_ulp=0.5`.

**CPE (cycles per element):** измеряется на горячем цикле, который многократно вызывает `logf8_avx2` на массиве длины кратной 8. Удобно взять `perf stat -e cycles` и разделить число циклов на **общее число вычисленных** `float` (итераций × 8). Для воспроизводимости — `-O3`, фиксированный размер массива, число повторов, модель CPU и закреплённая частота (performance governor).
