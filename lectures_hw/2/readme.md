
# Задание

На лекции были рассмотрены разные примеры бенчмаркинга. Требуется для рассмотренных или иных бенчмарков продемонстрировать, как адресуются проблемы по:
- репрезентативности нагрузок
- таймингу
- размерностям задач

# MLPerf

## Репрезентативность нагрузок

Тестовая нагрузка бенчмарков должна соответствовать реальным сценариям эксплуатации технологии, что проверяется специальными наборами тестов. В MLPerf представлены разные варианты прикладных задач, например связанных с классификацией объектов или работой с текстами. Для обучения и тестирнования используются общепринятые для определенных областей машинного обучения датасеты такие как **ImageNet**, **COCO**, **VOC** и т.д. Чтобы нагрузки были репрезентативными тесты бенчмарка покрывают почти весь процесс развертывания системы, начиная от железа и OC, заканчивая библиотеками и анализом архитектур сетей.

## Тайминг

Результаты работы систем должны быть воспроизводимы в условиях реального использования, поэтому возникает проблема точного измерения таймингов работы в тестовом режиме. С целью определения характерного времени работы введены сценарии: **single-stream** (ориентация на задержку, например для мобильных устройств), **server** (ориентация на пропускную способность, например для ЦОД), **multi-stream** (ориентация на смешанных нагрузок). Измерения таймингов по причине различных требований также разделено на стадии **тренировки** и **прогноза**.

## Размерность задач

Оптимальный бенчмарк должен быть масштабируем для систем разного размера и мощности. В бенчмарке MLPerf представлены разнообразные модели для разных задах, как легких так и тяжелых. Также бенчмарк обеспечивает возможность тестирования задач для разного оборудования: **GPU**, **TPU**, **CPU** и т.д. Сценарии из прошлого пункта помогают масштабировать нагрузку для систем разного размера. Помимо всего, тестирование поддерживает большое количество инструментов таких как **TensorFlow**, **PyTorch**, **CUDA** и т.д.
